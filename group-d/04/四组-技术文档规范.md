## 📋 执行摘要

## 附录：设计规范

### A.1 主色调分析

### 1.1 品牌主色

| 色彩类型 | HEX值 | RGB值 | 应用场景 |
|---------|------|-------|---------|
| **主色 - 品牌红** | `#E72521` | `rgb(231, 37, 33)` | CTA主要按钮（个人登录、申购等）、强调元素、重点提示 |
| **副色 - 企业蓝** | `#004098` | `rgb(0, 64, 152)` | 导航链接、次要按钮（机构登录、查看更多）、交互元素 |
| **页脚背景色** | `#0B264A` | `rgb(11, 38, 74)` | 页脚背景、深色区域 |

**色彩寓意：**
- **红色（#E72521）**：代表活力、信心、紧迫性，引导用户进行重要操作
- **深蓝（#004098）**：象征专业、信任、稳定，符合金融行业特质
- **深墨蓝（#0B264A）**：提供深层背景，增强品牌统一性

### 1.2 中性色系

| 色彩名称 | HEX值 | RGB值 | 应用场景 |
|---------|------|-------|---------|
| 纯白 | `#FFFFFF` | `rgb(255, 255, 255)` | 主要背景、卡片背景、文本背景（占比 60%+） |
| 深灰（标题） | `#333333` | `rgb(51, 51, 51)` | 标题、标签文本、重要信息 |
| 浅灰（正文） | `#222222` | `rgb(34, 34, 34)` | 正文、描述文本、主要内容 |
| 中灰（辅助） | `#888888` | `rgb(136, 136, 136)` | 辅助文本、时间戳、次要信息 |
| 浅背景灰 | `#EFF4F9` | `rgb(239, 244, 249)` | 内容区域背景、模块分割 |
| 极浅蓝 | `#E5F8FD` | `rgb(229, 248, 253)` | 信息提示区域背景 |

**配色特点：**
- 使用灰度系列实现视觉层级感
- 高对比度确保可读性
- 浅色背景营造简洁、专业的氛围

## 2. 字体样式分析

### 2.1 字体族群

**系统字体堆栈（从上到下依次调用）：**
```
-apple-system, 
"system-ui", 
"Segoe UI", 
system-ui, 
Roboto, 
Ubuntu, 
Cantarell, 
"Noto Sans", 
sans-serif, 
"PingFang SC",        /* 苹果苹方体 - 中文首选 */
"Microsoft YaHei",    /* 微软雅黑 - Windows中文 */
"Helvetica Neue", 
Helvetica, 
Arial
```

**特点：** 
- 采用**系统字体优先**策略，无需加载外部字体文件
- Mac系统优先使用"PingFang SC"（苹方），提升视觉效果
- Windows系统使用"Microsoft YaHei"（微软雅黑），保证中文显示
- 降级方案确保各平台的最佳显示效果

### 2.2 标题字体样式

| 标题等级 | 字号 | 字重 | 行高 | 颜色 | 下边距 |
|---------|------|------|------|------|--------|
| **H1** | 28px | 500 | 30.8px | `#FFFFFF` | 18.76px |
| **H2** | 30px | 500 | 33px | `#333333` | 20px |
| **H3** | 24px | 500 | ~26.4px | `#333333` | ~16px |

**特点：**
- 字重统一为 **500**（Medium），视觉柔和但有分量感
- 行高为字号的 **1.1-1.15倍**，便于阅读
- H1使用白色，通常出现在深色背景hero区域
- H2/H3使用深灰，确保内容区的可读性

### 2.3 正文字体样式

| 字体类型 | 字号 | 字重 | 行高 | 颜色 | 应用 |
|---------|------|------|------|------|------|
| **正文** | 18px | 400 | 正常 | `#222222` | 长文本内容 |
| **小文本** | 14px | 400 | 正常 | `#888888` | 时间、标签、次要信息 |
| **链接文本** | 16px | 400 | 正常 | `#004098` | 导航、CTA链接 |

**特点：**
- 正文字号为**18px**，相对较大，提升阅读体验
- 行高设置合理，文本密集度适中
- 字重均为**400**（Regular），保持内容的中立性

### 2.4 按钮文本

| 按钮类型 | 字号 | 字重 | 行高 |
|---------|------|------|------|
| 主要按钮 | 16px | 400 | 48px |
| 次要按钮 | 14px | 400 | 正常 |

```
│  - 内容两侧间距: 131.67px               │ 内边距: 60px
│                                         │
│  内容模块 (模块1, 模块2, ...)           │
│  - 公司简介、投研体系、ESG投资          │ 每个内间距: 60px

│  页脚 (Footer)                          │
│  - 多列链接导航                         │ 背景: #0B264A
│  - 社交媒体                             │ 内边距: 90px 0 60px
│  - 备案信息                             │
└─────────────────────────────────────────┘
```

### 3.2 间距系统

**水平间距：**
- **页面两侧外边距：** 131.67px（约占屏幕宽度的12-15%）
- **模块内部内边距：** 60px
- **元素间距：** 20px, 30px, 40px（递进式）

**垂直间距：**
- **section间距：** 60px
- **模块间距：** 40px, 80px
- **元素间距：** 20px, 30px

**特点：**
- 采用**对称的页面边距**，实现内容居中
- 左右对称，视觉平衡
- 足量的负空间提升整体品质感

### 3.3 容器设计

| 区域类型 | 宽度 | 背景 | 边框 | 圆角 | 投影 |
|---------|------|------|------|------|------|
| 主容器 | 100% | 变量 | 无 | 0px | 无 |
| 卡片 | 100% | 白色/浅灰 | 无 | 0px | 无 |
| 按钮 | 自适应 | 实色 | 无 | 8px | 无 |
| 输入框 | 100% | 白色 | 1px solid #DDD | 4px | 无 |

**特点：**
- 采用**扁平设计**，无任何投影效果
- 使用**纯色背景**区分内容层次
- **直角或极小圆角**，体现现代感

## 5. 图片风格分析

### 5.1 图片分类与尺寸

| 图片类型 | 尺寸 | 风格 | 应用 |
|---------|------|------|------|
| **LOGO** | 121×88px | 彩色品牌标识 | 页头、页脚 |
| **App宣传图** | 120×120px | 纯色/渐变 + 图标 | App下载推广 |
| **导航悬停图** | 201×108px - 402×216px | 插画/实景 | 菜单悬停显示 |
| **功能图标** | 16×16px - 48×48px | 单色线条或纯色 | 功能按钮、装饰 |
| **二维码** | 128×128px | 标准黑白二维码 | 微信、App下载 |

### 5.2 图片色调处理

**色调特点：**
1. **深蓝主调** - 页面各处装饰图片以深蓝为基调
2. **高饱和度** - 色彩鲜明，提高视觉冲击力
3. **简化处理** - 避免复杂细节，强调关键信息
4. **渐变应用** - 部分图片使用蓝白渐变

**滤镜应用：**
- 部分导航图片可能应用了**轻微的模糊**或**对比度调整**
- 主要通过**不透明度控制**（80-95%）实现图层效果

### 5.3 图片布局

**Hero背景：**
- 方向：从左到右
- 覆盖方式：全幅背景
- 内容覆盖：文案居中或左侧
- 响应式：图片随视口调整

**内容图片：**
- 位置：内容旁侧或上方
- 宽度：100%容器宽度或特定尺寸
- 对齐：与文案对齐

## 8. 视觉设计特色总结

### 8.1 设计风格特征

✅ **扁平化设计**
- 无投影、无渐变、纯色构成
- 现代、清爽、专业

✅ **高对比度**
- 深色文本 vs 浅色背景
- 红色按钮 vs 蓝色链接
- 确保可读性和视觉导向

✅ **冷色调主导**
- 蓝色系是主要色调
- 营造专业、值得信赖的感觉
- 适合金融行业特征

✅ **充足的负空间**
- 页面边距宽裕（131.67px）
- 模块间距大（60px）
- 提升整体品质感

✅ **系统化设计**
- 统一的字体、颜色、间距
- 遵循设计规范
- 保证视觉一致性

### 8.2 用户体验考量

- **清晰的信息层级**：通过字号、颜色、位置引导视线
- **易识别的操作**：红色按钮一眼即明白是主操作
- **响应式友好**：大间距、大文字便于各设备浏览
- **快速加载**：避免复杂效果，使用简单几何形状
- **易于导航**：固定导航栏，多层级菜单结构清晰

## 9. 可用性建议

### 针对设计师

1. **严格遵循颜色规范**：使用规定的品牌色，避免自行调配
2. **保持字体一致**：所有中文内容使用系统字体栈
3. **间距要规范**：使用60px、40px、20px等标准间距
4. **按钮要明显**：红色用于主操作，蓝色用于次操作
5. **图片要统一**：色调处理应保持一致性

### 针对开发者

1. **使用CSS变量**存储品牌色，便于全局修改
2. **建立可复用的组件库**（按钮、卡片、表单等）
3. **优先使用系统字体**，减少外部字体加载
4. **验证色彩对比度**，确保WCAG可访问性标准
5. **响应式设计**：保持各设备上的间距比例
