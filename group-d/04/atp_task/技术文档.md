## 1. 项目概述

### 1.1 项目背景

网址导航管理系统是一个基于 Flask 框架的 Web 应用，旨在为用户提供便捷的网址分类管理和快速访问功能。系统支持多版本管理、分类折叠、网址增删改查等功能，适用于企业内部办公网址导航或个人网址收藏管理。

### 1.2 技术栈

| 技术类别 | 技术选型 | 版本 |
|---------|---------|------|
| 后端框架 | Flask | 3.0.3 |
| WSGI服务器 | Waitress | 3.0.0 |
| 跨域支持 | Flask-CORS | 5.0.0 |
| 前端样式 | Tailwind CSS | CDN |
| 图标库 | Font Awesome | 6.5.1 |
| 数据存储 | JSON 文件 | - |

### 1.3 术语定义

| 术语 | 定义 |
|-----|------|
| 版本 | 一组网址配置的集合，支持多版本切换 |
| 分类 | 网址的分组类别，支持自定义排序和折叠 |
| 网址项 | 包含标题、URL、图标的单个网址记录 |
| 当前版本 | 用户当前正在查看和编辑的版本 |

## 2. 快速开始

### 2.1 环境要求

- Python 3.8+
- Windows/Linux/macOS
- 端口 80 可用（或修改配置）

### 2.2 安装步骤

```bash
# 1. 克隆或解压项目到本地目录
cd atp_task

# 2. 安装依赖
pip install Flask==3.0.3 Flask-CORS==5.0.0 requests python-dotenv selenium waitress

# 3. 启动服务
python app.py
```

### 2.3 访问应用

服务启动后，通过以下地址访问：

- 首页：http://localhost/
- 网址导航：http://localhost/get_my_website

## 4. API参考

### 4.1 接口规范

- **基础URL**: `http://localhost`
- **数据格式**: JSON
- **编码**: UTF-8

### 4.2 接口列表

#### 4.2.1 获取网址数据

```http
GET /get_my_website/data?version_id={version_id}
```

**参数**:
| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| version_id | string | 否 | 版本ID，默认为 default |

**响应示例**:
```json
{
  "versions": [
    {
      "id": "default",
      "name": "默认版本",
      "urls": [...],
      "collapsedCategories": [],
      "categoryOrder": []
    }
  ],
  "officeCurrentVersionId": "default",
  "current_version": {...}
}
```

#### 4.2.2 保存网址数据

```http
POST /get_my_website/data
```

**请求体**:
```json
{
  "version_id": "default",
  "current_version": {
    "id": "default",
    "name": "默认版本",
    "urls": [...]
  }
}
```

#### 4.2.3 添加新网址

```http
POST /get_my_website/urls
```

**请求体**:
```json
{
  "version_id": "default",
  "title": "网址标题",
  "url": "https://example.com",
  "icon": "fa-globe",
  "category": "常用网站"
}
```

#### 4.2.4 更新网址

```http
PUT /get_my_website/urls/{url_id}
```

#### 4.2.5 删除网址

```http
DELETE /get_my_website/urls/{url_id}?version_id={version_id}
```

#### 4.2.6 创建新版本

```http
POST /get_my_website/create_version
```

**请求体**:
```json
{
  "name": "新版本名称",
  "sourceId": "default"
}
```

#### 4.2.7 切换版本

```http
POST /get_my_website/switch_version/{version_id}
```

### 4.3 错误码

| 状态码 | 说明 |
|-------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 5. 前端界面

### 5.1 界面说明

系统包含两个主要页面：

| 页面 | 路由 | 功能描述 |
|-----|------|---------|
| 首页 | `/` | 测试管理平台入口 |
| 网址导航管理 | `/get_my_website` | 网址分类管理、版本切换 |

### 5.2 核心功能

#### 版本管理
- 创建新版本（基于现有版本复制）
- 切换当前版本
- 版本数据隔离

#### 网址管理
- 添加网址（指定分类、图标、URL）
- 编辑网址信息
- 删除网址
- 按分类展示

#### 分类管理
- 分类折叠/展开
- 分类排序
- 自定义分类名称

### 5.3 技术实现

- **样式框架**: Tailwind CSS (CDN)
- **图标库**: Font Awesome 6.5.1 (CDN)
- **交互**: 原生 JavaScript
- **响应式**: 支持移动端适配

## 7. 部署运维

### 7.1 部署方案

#### Windows 部署

使用批处理脚本启动：

```batch
@echo off
cd /d C:\path\to\atp_task
python app.py
```

或使用 Waitress 生产服务器（已集成在 app.py 中）

#### Linux 部署

```bash
# 使用 systemd 服务
sudo systemctl enable atp-task
sudo systemctl start atp-task
```

### 7.2 监控告警

- 日志监控：检查 `logs/app.log` 错误信息
- 端口监控：确保 80 端口正常监听
- 磁盘监控：JSON 数据文件大小

### 7.3 故障处理

| 问题 | 解决方案 |
|-----|---------|
| 端口被占用 | 修改环境变量 PORT 或使用其他端口 |
| 模块导入错误 | 检查 Python 路径，使用相对导入 |
| JSON 数据损坏 | 备份恢复或重建配置文件 |
| 权限不足 | 以管理员身份运行或修改目录权限 |

## 附录：设计规范

### A.1 主色调分析

#### 品牌主色

| 色彩类型 | HEX值 | RGB值 | 应用场景 |
|---------|------|-------|---------|
| **主色 - 品牌红** | `#E72521` | `rgb(231, 37, 33)` | CTA主要按钮、强调元素、重点提示 |
| **副色 - 企业蓝** | `#004098` | `rgb(0, 64, 152)` | 导航链接、次要按钮、交互元素 |
| **页脚背景色** | `#0B264A` | `rgb(11, 38, 74)` | 页脚背景、深色区域 |

**色彩寓意：**
- **红色（#E72521）**：代表活力、信心、紧迫性，引导用户进行重要操作
- **深蓝（#004098）**：象征专业、信任、稳定，符合管理类系统特质
- **深墨蓝（#0B264A）**：提供深层背景，增强品牌统一性

#### 中性色系

| 色彩名称 | HEX值 | RGB值 | 应用场景 |
|---------|------|-------|---------|
| 纯白 | `#FFFFFF` | `rgb(255, 255, 255)` | 主要背景、卡片背景、文本背景 |
| 深灰（标题） | `#333333` | `rgb(51, 51, 51)` | 标题、标签文本、重要信息 |
| 浅灰（正文） | `#222222` | `rgb(34, 34, 34)` | 正文、描述文本、主要内容 |
| 中灰（辅助） | `#888888` | `rgb(136, 136, 136)` | 辅助文本、时间戳、次要信息 |
| 浅背景灰 | `#EFF4F9` | `rgb(239, 244, 249)` | 内容区域背景、模块分割 |
| 极浅蓝 | `#E5F8FD` | `rgb(229, 248, 253)` | 信息提示区域背景 |
