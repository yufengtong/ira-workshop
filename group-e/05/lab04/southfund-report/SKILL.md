# 南方基金风格技术方案报告生成器

## 概述

同时生成两个文件：**HTML 报告**（主交付物，视觉效果完整）和 **Markdown 报告**（版本管理友好）。两者内容一致，HTML 为最终呈现格式。

## 工作流程

1. **收集信息**：确认项目名称、英文副标题、文档编号、版本号、编制单位、编制日期、密级
2. **确定章节**：与用户确认报告的目录章节（通常 8-10 章）
3. **生成 HTML**：基于 [html-template.html](html-template.html) 模板结构生成
4. **生成 Markdown**：基于 [md-template.md](md-template.md) 模板结构生成
5. **验证**：确认文件可正常打开

## 设计规范

### 色彩系统

| 变量 | 色值 | 用途 |
|------|------|------|
| --primary-blue | #004098 | 主色，标题、表头、边框 |
| --dark-blue | #00295F | 封面渐变起点、strong 文字 |
| --light-blue | #1A6FD5 | 封面渐变终点、链接 |
| --bg-blue | #E8EFF8 | 页面背景、引用块背景 |
| --border-blue | #99B8D9 | 表格边框、代码块边框 |
| --accent-red | #E72521 | 警告块、重要声明边框 |
| --accent-gold | #C9A96E | 预留装饰色 |

### HTML 页面结构（5 页分页）

```
Page 1: 封面（渐变背景 + 项目信息表） + 免责声明横幅
Page 1: 版本记录 + 审批记录 + 文档分发 + 目录
─── page-break ───
Page 2: 第1-2章（背景需求 + 建设目标）
─── page-break ───
Page 3: 第3章（技术架构，含架构图）
─── page-break ───
Page 4: 第4-5章（功能设计 + 多Agent协作/核心模块）
─── page-break ───
Page 5: 第6-10章（安全合规 + 部署 + 风险 + 实施计划 + 总结）
尾部: 再次声明 + 页脚
```

### 关键 HTML 组件

**封面**：`div.cover` 渐变背景 + `table.cover-table` 项目元数据

**分页符**：`<hr class="page-break">` 在屏幕显示为渐变线，打印时分页

**章节标题**：`<h2 class="section-title">` 蓝色下划线

**蓝色高亮块**：`div.styled-block` 左侧蓝色竖线 + 渐变背景
- 红色变体：`div.styled-block.red` 用于重要警告章节

**表格**：蓝色表头 + 斑马纹 + 悬停效果，`table.meta-table` 用于封面区小表

**ASCII 架构图**：`<pre>` 深色背景，用 box-drawing 字符绘制

**目录框**：`div.toc-box` 渐变背景 + 有序列表

**免责声明**：`div.disclaimer-banner` 黄色警告 + `blockquote` 红色左边框

**页脚**：`div.page-footer` 深蓝背景三栏布局

### Markdown 对应写法

| HTML 组件 | Markdown 写法 |
|-----------|--------------|
| 封面 | `<div align="center" style="...">` 内嵌 HTML |
| 分页符 | `---` 水平线 |
| styled-block | `<div style="border-left: 4px solid #004098; ...">` |
| 表格 | 标准 Markdown 表格 |
| 架构图 | 三个反引号代码块 |
| 目录框 | `<div style="...">` + 有序列表 |
| 免责声明 | `> ⚠️ **免责声明**：...` |

## 内容规范

### 必选章节（可根据项目调整）

1. 项目背景与需求分析
2. 项目建设目标（含功能树）
3. 技术架构方案（含选型表 + ASCII 架构图）
4. 核心功能设计
5. [项目特有核心模块]（如多智能体、数据管道等）
6. 安全与合规保障
7. 部署实施方案（含 CI/CD 表）
8. 风险评估与应对（技术/合规/运维三维度）
9. 项目实施计划（含里程碑表）
10. 总结与展望

### 写作风格

- 语言：中文，正式技术文档风格
- 段落：`text-align: justify`，14.5px 字号
- 要点用表格或列表呈现，避免大段纯文字
- 架构图、流程图使用 ASCII box-drawing 字符在 `<pre>` 中呈现
- 每章开头使用 `styled-block` 包裹概述段落
- 涉及金融/投资的报告必须包含免责声明

### 封面元数据模板

```
项目名称: [项目全称]
文档编号: [项目代号]-TECH-[年份]-[序号]
版　　本: V1.0
密　　级: 内部公开
编制单位: [部门名称]
编制日期: [年月]
```

## 输出要求

1. HTML 文件：完整独立的 `.html` 文件，所有样式内联在 `<style>` 中，无外部依赖
2. Markdown 文件：对应的 `.md` 文件，使用内嵌 `<style>` 标签提供主题色
3. 文件命名：`{项目名称}技术方案报告.html` 和 `{项目名称}技术方案报告.md`
4. 编码：UTF-8，`lang="zh-CN"`
