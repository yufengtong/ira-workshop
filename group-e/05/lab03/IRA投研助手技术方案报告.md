## 1. 项目背景与需求分析

<div style="border-left: 4px solid #004098; padding-left: 15px; background: linear-gradient(to right, #E8EFF8, transparent); margin: 10px 0;">

### 1.1 行业背景

随着资产管理行业数字化转型深入，投研团队面临日益增长的信息处理压力。传统投研模式存在以下痛点：

- **信息过载**：海量研报、新闻、行情数据需要人工筛选与整合，效率低下
- **观点单一**：单人研究员视角有限，难以全面覆盖看多/看空多维度分析
- **合规风险**：人工撰写的投研报告可能触发监管红线（禁荐股、保本承诺等）
- **溯源困难**：投研结论缺乏系统化的数据来源追踪与审计链路
- **多源整合难**：行情、研报、舆情等多源数据分散在不同系统，缺乏统一编排

### 1.2 需求分析

基于上述背景，本项目旨在构建一套**智能投研助手系统**，满足以下核心需求：

| 需求类别 | 具体需求 |
|---------|---------|
| 多源数据整合 | 统一接入行情快照、研报知识库、舆情新闻等多源数据 |
| 多智能体协作 | 支持看多/看空/纪要多角色 Agent 结构化辩论与交叉验证 |
| 合规闸门 | 内置规则引擎，自动扫描并过滤违规话术与不当建议 |
| 全链路溯源 | 每条分析结论可追溯至原始数据源（chunk ID / 时间戳） |
| 投研报告生成 | 自动生成结构化投研纪要，支持导出与审计 |
| CoPaw底座集成 | 基于 CoPaw 多代理框架实现 Agent 编排、技能扩展与渠道接入 |

| 技术项 | 选型方案 | 选型理由 |

## 5. 多智能体协作体系

<div style="border-left: 4px solid #E72521; padding-left: 15px; background: linear-gradient(to right, #FDE8E8, transparent); margin: 10px 0;">

### 5.1 设计原则

- **人机边界**：多 Agent 仅辅助信息整理与观点对照，**不构成投资建议**，不替代合规与投委会决策
- **可溯源**：凡引用行情、新闻、研报片段，须标注来源（工具回执 ID / chunk ID / 时间戳）
- **工具最小化**：每 Agent 仅允许白名单工具，默认**只读**，禁止下单、改仓、代客操作类工具

### 5.2 Agent 角色定义（P0 三角色闭环）

| Agent ID | 展示名 | 角色定位 | 论证立场 | 允许工具 | 显式禁止 |
|----------|--------|---------|---------|---------|---------|
| M5-BULL | 看多研究员 | 从成长与催化剂角度补充论据 | 建设性、偏乐观 | 只读行情(M2)、知识库检索(M3) | 编造数值；承诺收益；诱导交易 |
| M5-BEAR | 风控/看空研究员 | 从估值与下行风险角度质疑 | 质疑性、偏谨慎 | 只读行情(M2)、知识库检索(M3) | 编造数值；承诺收益；诱导交易 |
| M5-MOD | 辩论主持/纪要官 | 不站队；结构化归纳分歧 | 中立 | 格式化/摘要（禁止拉取新行情） | 输出买卖建议；替用户做决策 |

### 5.3 辩论编排流程

```
   BFF 请求          编排器广播         R1 并行分析        R2 交叉验证        合规闸门          纪要输出
  ┌────────┐      ┌────────────┐      ┌──────────┐      ┌──────────┐      ┌──────────┐      ┌────────┐
  │ symbol │ ──→  │ fan_out    │ ──→  │ BULL     │ ──→  │ BEAR回应 │ ──→  │ R-G01~04 │ ──→  │ MOD    │
  │ + mode │      │ 3 Agents   │      │ BEAR     │      │ BULL补充 │      │ 规则扫描  │      │ 汇总   │
  └────────┘      └────────────┘      │ MOD旁听  │      └──────────┘      └──────────┘      └────────┘
                                      └──────────┘
```
