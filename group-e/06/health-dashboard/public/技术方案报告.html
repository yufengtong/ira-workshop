1. 项目背景与需求分析

1.1 业务背景

在现代电商系统中，用户体验的流畅性直接影响业务转化率。从用户登录到完成支付的完整链路中， 任何一个环节出现问题都可能导致用户流失。因此，需要一套实时监控系统来跟踪关键业务流程的健康状态。

1.2 需求分析

需求类别 具体需求
流程可视化 直观展示用户旅程各步骤的状态
实时监控 显示自动化测试通过率和响应时间
状态标识 红绿灯标识健康状态（健康/警告/严重）
异常告警 链路断裂时自动高亮异常节点

2. 项目建设目标

项目建设目标
✓ 流程可视化：清晰展示登录→搜索→加购→下单→支付完整链路
✓ 实时监控：每5秒自动刷新测试通过率和响应时间
✓ 智能告警：通过率<70%或响应时间>1000ms自动标记为严重
✓ 链路追踪：异常节点自动高亮，快速定位问题
✓ 直观展示：红绿灯标识 + 动画效果，一目了然

4.2 健康状态判定规则

状态 颜色 判定条件 说明
健康 绿色 通过率≥90% 且 响应时间≤500ms 正常运行
警告 黄色 通过率70-90% 或 响应时间500-1000ms 需要关注
严重 红色 通过率<70% 或 响应时间>1000ms 链路断裂

4.3 数据模型设计

// 健康状态枚举
enum HealthStatus {
  HEALTHY = 'healthy',      // 绿色 - 健康
  WARNING = 'warning',      // 黄色 - 警告
  CRITICAL = 'critical',    // 红色 - 严重
}

// 步骤指标数据
interface StepMetrics {
  stepId: string;            // 步骤ID
  passRate: number;          // 测试通过率 (%)
  avgResponseTime: number;   // 平均响应时间 (ms)
  totalTests: number;        // 总测试数
  passedTests: number;       // 通过测试数
  lastUpdated: Date;         // 最后更新时间
}

// 步骤健康状态
interface StepHealth {
  step: BusinessStep;        // 步骤信息
  metrics: StepMetrics;      // 指标数据
  status: HealthStatus;      // 健康状态
  isBroken: boolean;         // 是否链路断裂
}
