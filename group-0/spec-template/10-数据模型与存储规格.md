## 1. 概念模型

```
Session（会话）1 ── * QARecord（问答记录）
   │                      │
   │  session_id (PK)     │  id (PK)
   │  title               │  session_id (FK → Session)
   │  query_count          │  query / answer / llm_used
   │  created_at           │  response_time_ms / answer_source
   └  updated_at           └  timestamp
```

> **核心关系**：一个 Session 可包含零到多条 QARecord。外键通过 `session_id` 字段逻辑引用。

## 2. 存储引擎

| 项 | 说明 |
|----|------|
| 引擎 | JSON 文件（`json` 标准库） |
| 编码 | UTF-8，缩进 2 空格 |
| 读写模式 | RMW（全量读入 → 修改 → 全量写回） |
| 并发安全 | 无锁，不支持并发写入 |
| 文件布局 | `{DATA_DIR}/sessions.json` + `qa_records.json` |

## 3. 实体定义 — Session

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `session_id` | `string` (UUID) | 是 | **PK** | 会话唯一标识 |
| `title` | `string` | 是 | ≤23 字符 | 默认"新会话"，首次问答自动重命名 |
| `created_at` | `string` (ISO-8601) | 是 | UTC+Z | 创建时间 |
| `updated_at` | `string` (ISO-8601) | 是 | UTC+Z | 最后更新时间 |
| `query_count` | `integer` | 是 | ≥ 0 | 累计问答次数 |

## 4. 实体定义 — QARecord

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `id` | `string` | 是 | **PK**，`rec_{ts}` | 基于时间戳生成 |
| `session_id` | `string` | 是 | **FK** → Session | 所属会话 |
| `query` | `string` | 是 | 1–500 字符 | 用户提问原文 |
| `answer` | `string` | 是 | — | LLM/Demo 返回的答案 |
| `llm_used` | `boolean` | 是 | RULE-005 | 是否使用真实 LLM |
| `model` | `string\|null` | 是 | — | 模型标识 |
| `response_time_ms` | `integer` | 是 | SLO < 5000 | 响应耗时（ms） |
| `answer_source` | `string\|null` | 否 | 枚举 | `copaw`\|`bailian`\|`demo` |

## 6. 关键业务逻辑

**示例 — 首次问答自动命名**：

```python
if session["query_count"] == 1:
    session["title"] = query[:20] + ("..." if len(query) > 20 else "")
```

> **规则**：仅首次问答（query_count 从 0 → 1）触发，后续不再自动更名。

| 逻辑 | 触发条件 | 行为 |
|------|----------|------|
| 首次问答自动命名 | `query_count` 从 0 → 1 | 标题 = query 前 20 字 + `...` |
| 级联删除 | 删除 Session | 同步删除所有关联 QARecord |
| 【请补充其他业务逻辑】 | 【触发条件】 | 【行为】 |
