一、项目概述

1.1 项目背景

在企业数字化转型过程中，数据驱动决策已成为核心竞争力。然而，传统数据查询方式存在技术门槛高、查询效率低、数据可读性差等突出问题。业务人员需依赖技术团队编写SQL语句获取数据，从需求提出到数据交付往往需要数小时甚至数天，严重制约了业务决策效率。

智能问数系统（Intelligent Query System，以下简称IQS）旨在通过自然语言处理（NLP）和AI大模型技术，构建一个面向全员的数据查询与分析平台。用户只需以日常对话的方式提问，系统即可自动理解语义、检索数据、生成分析结果，并以文字描述和可视化表格的形式直观呈现，真正实现"人人都是数据分析师"的愿景。

1.2 项目目标

表 1-1 项目目标矩阵
目标维度 具体描述 衡量指标
降低门槛 支持自然语言查询，消除SQL技术壁垒 非技术人员自助查询率≥85%
提升效率 即时响应数据查询需求，缩短数据获取周期 平均查询响应时间<3秒
精准分析 AI辅助数据解读，提供趋势分析和异常预警 查询结果准确率≥95%
安全合规 细粒度数据权限控制，保障数据安全 数据泄露事件为零
用户体验 提供流畅的对话式交互体验 用户满意度≥4.5/5

1.3 系统定位

产品定位：AI驱动的企业级数据查询助手
  • 目标用户：企业全员，包括业务人员、管理层、运营团队等非技术角色
  • 核心能力：自然语言理解 → 智能SQL生成 → 数据查询执行 → 结果可视化呈现
  • 交互模式：对话式Chat UI，支持上下文关联的多轮对话
  • 数据范围：覆盖销售、用户、库存、财务、运营等核心业务域

4.2 自然语言理解模块

自然语言理解模块是系统的核心智能组件，负责将用户的自然语言问题转化为结构化的查询意图。

4.2.1 意图识别

表 4-1 查询意图分类
意图类别 描述 示例 关键词提取
指标查询 查询某一具体指标的数值 "上月销售额是多少" 时间=上月, 指标=销售额
趋势分析 分析指标的变化趋势 "用户增长趋势如何" 对象=用户, 分析=趋势
对比分析 比较不同维度的数据差异 "各区域销售对比" 维度=区域, 指标=销售
异常检测 识别数据中的异常和预警 "哪些商品库存不足" 对象=商品, 条件=库存不足
汇总统计 聚合统计某类数据 "退货退款数据汇总" 对象=退货退款, 操作=汇总

4.2.2 实体抽取

系统通过NER（命名实体识别）技术从用户问题中提取关键业务实体：

  • 时间实体："上个月"、"第一季度"、"2026年3月"等时间表达
  • 指标实体："销售额"、"利润"、"退货率"等业务指标
  • 维度实体："区域"、"品类"、"渠道"等分析维度
  • 条件实体："增长超过10%"、"库存不足"等筛选条件

4.3 智能SQL生成模块

基于LLM大语言模型，将结构化的查询意图转化为可执行的SQL语句。核心设计如下：

⚠️ 安全约束

SQL生成模块必须遵循严格的安全规范：仅允许SELECT查询操作，禁止INSERT/UPDATE/DELETE等写入操作；所有生成的SQL需经过语法校验和安全审计后方可执行；执行超时上限为30秒，防止慢查询影响系统性能。

4.3.1 SQL生成流程

1. 接收结构化查询意图（意图类别 + 实体列表）
2. 匹配数据库元数据（表名、字段名、关联关系）
3. 调用LLM生成候选SQL语句
4. SQL安全校验（白名单过滤、注入检测）
5. SQL语法验证（EXPLAIN预检）
6. 执行查询并返回结果集

4.4.1 结果处理策略

表 4-2 数据呈现策略
数据特征 呈现方式 说明

4.4.2 数据格式化规范

数据展示规范
  • 金额数据：千分位分隔，保留一位小数，单位"万元"，如 1,283.6 万元
  • 百分比数据：保留一位小数，正值前缀"+"并标绿，负值标红，如 +12.4%
  • 数量数据：千分位分隔，单位明确，如 12,680 单
  • 状态标识："正常"绿色显示，"预警"红色显示，"达标"绿色显示

6.3 SQL注入防护

⚠️ SQL安全防护策略
  • 白名单机制：仅允许SELECT语句，拦截所有DDL/DML操作
  • 关键字过滤：禁止 DROP、DELETE、UPDATE、INSERT、ALTER、TRUNCATE 等危险关键字
  • 参数化查询：所有动态值通过参数化方式传入，杜绝拼接注入
  • 执行限制：单次查询超时30秒自动终止，结果集上限10000行
  • 审计追溯：所有生成的SQL及执行结果完整记录，支持事后审计
