Section 01

项目概述

Wiki-Plus 是一个以 "LLM Wiki 模式" 为核心的个人知识管理与学习笔记系统，旨在通过 Claude 与 Obsidian 的协作，构建持久、可复利增长的知识库。该项目强调 "人类策划源 + LLM 维护" 的理念：人类负责选择高质量的来源并提出问题，LLM 负责阅读、整合、维护与优化知识结构。

核心价值

⚙
降低维护成本
通过自动化维护减少跨引用、矛盾标注与页面更新的人工负担。
⚡
提升查询效率
预编译的知识结构与热缓存使会话启动更快、token 成本更低。
✓
适合中小规模
在页面数量处于百级到千级时，无需向量化基础设施即可获得良好体验。

适用场景与目标用户

  • 个人第二大脑与终身学习者 — 构建可持续增长的知识资产
  • 研究助理与内容整理者 — 高效整理和检索多源信息
  • 小团队知识沉淀 — 跨项目上下文共享与协作
  • 低成本知识检索与合成场景 — 无需复杂基础设施

项目结构

项目采用"知识库驱动 + 多层协作"的组织方式，围绕三个核心目录展开：

目录职责说明
.raw/ 源文档层 不可变的原始源文档，存放 PDF、Markdown、转录等
wiki/ 知识库层 LLM 生成的知识库，包含 concepts/、entities/、sources/、meta/ 等
CLAUDE.md 指令层 定义知识库结构、命名规范与维护流程，指导 LLM 如何工作
skills/ 技能集合 Agent 技能定义，包含 wiki、wiki-ingest、wiki-query 等
hooks/ 自动化 会话钩子，驱动热缓存注入与自动提交

热缓存机制

Hot Cache (wiki/hot.md)

约 500 字的会话上下文摘要，包含"最近更新""关键事实""近期变化""活跃主题"。通过 SessionStart 钩子在会话开始时自动注入，显著降低每次会话的 token 成本。

热缓存工作流程

会话开始
→
读取 hot.md
→
命中上下文？
→
直接回答 / 读取索引
→
更新热缓存
  • 更新时机：每次摄取后、重要查询后、会话结束时
  • 格式规则：固定 frontmatter + 标题，严格控制字数与覆盖写入
  • 会话注入：SessionStart 钩子自动读取，PostCompact 钩子压缩后恢复

数据流处理

摄取流水线

1. 差异检查
通过 .raw/.manifest.json 检查源文件哈希，命中则跳过
2. 源获取与清洗
WebFetch 获取 URL 内容，可选 defuddle 清洗；文件完全读取；图片 OCR 识别
3. 内容提取
创建源摘要页，提取实体与概念，建立交叉引用
4. 索引更新
更新 index.md、域子索引、log.md、hot.md 与 overview.md
5. 矛盾检测
若新信息与现有页面冲突，在双方添加矛盾 callout

组件交互关系

  • 指令层对摄取/查询/整理存在路由依赖，wiki 技能作为总控路由
  • 摄取与查询均依赖热缓存与索引
  • 钩子与自动化贯穿各流程，保障上下文一致性与版本控制
  • 模板系统为摄取提供标准化页面骨架

Section 04

知识管理

知识管理围绕 "智能摄取 — 查询检索 — 健康检查" 三大支柱构建闭环机制，确保知识库的高一致性、高可检索性与可持续演进。

智能摄取系统

多格式源处理

  • 文件摄取：完全读取源文件，讨论关键要点，创建源摘要与实体/概念页面
  • URL 摄取：WebFetch 获取，可选 defuddle 清洗，派生 slug 并保存
  • 图片/视觉：OCR 识别与描述，保存至 .raw/images
  • 批处理：逐个执行单源流程，结束后统一交叉引用传递

差异追踪与矛盾检测

  • 差异追踪：通过 .manifest.json 记录哈希，相同则跳过
  • 矛盾标记：新信息冲突时，在双方页面添加 contradiction callout
  • Token 纪律：优先热缓存与索引，限制读取页面数量，PATCH 精确编辑

查询检索系统

模式Token 成本读取范围适用场景
Quick ~1,500 仅 hot.md + index.md 快速事实、日期查找、简要说明
Standard ~3,000 hot + index + 3-5 页面 日常问题、概念解释、实体介绍
Deep ~8,000+ 完整 wiki + 可选网络 跨领域比较、综合分析、差距识别

索引与缓存策略

分层索引设计

主索引 (index.md) + 域子索引 (entities/_index.md, concepts/_index.md, sources/_index.md) + 热缓存 (hot.md) + 日志 (log.md) 构成知识库的导航与检索中枢。摄取后统一更新索引，查询时按层级逐步深入。

可视化与仪表板

Wiki-Plus 通过 "知识图谱 + 可视化画布 + 交互式仪表板" 三位一体的方式构建可视化体系，从"内容生产"到"可视化洞察"实现高效流转。

🖼
Canvas 画布
以 JSON Canvas 1.0 开放标准定义节点、分组区域与连接边，支持文本、文件、链接等节点类型，配合自动布局算法实现高效组织。
📊
Bases 仪表板
采用 Obsidian Bases 嵌入视图，结合过滤器、公式与多视图（表格、列表）实现"实时数据视图"与动态监控。
🌐
Graph View 图谱
按路径与类型着色，支持隐藏未解析/孤儿节点，提供全局关系浏览与导航。
🗺
Wiki Map 知识地图
以 hub 为中心的层级关系可视化，辅以边标签与颜色编码，实现"宏观图谱 + 微观画布"。

数据视图配置

视图类型适用场景关键要素
表格视图 近期活动、比较结果、源信息 过滤器、多字段排序、分组
列表视图 种子页面、缺失来源的实体、开放问题 条件过滤、按状态/时间排序

API 参考

Wiki-Plus 的 API 层由 "本地 REST API + MCP 工具链 + 本地服务 (OpenCode)" 构成，形成"文档读写 + 搜索 + 可视化 + 本地进程"的闭环。

REST API

基础信息

地址：https://127.0.0.1:27124  |  认证：Authorization: Bearer <API_KEY>

操作方法路径说明
读取文件GET/vault/{folder}/{path}读取 Markdown 文件内容
创建/替换PUT/vault/{folder}/{path}正文为 Markdown
追加内容POST/vault/{folder}/{path}追加 Markdown 内容
补丁PATCH/vault/{folder}/{path}编辑 frontmatter/标题段落
搜索POST/search/simple/?query={q}关键词搜索
标签列表GET/tags/列出所有标签

WebSocket 接口

  • 通过 OpenCodeClient 与本地 OpenCode 服务器实时交互
  • 支持状态变更回调与事件订阅 (starting / running / error / stopped)
  • UI 订阅状态变化，自动刷新界面

本地服务接口 (IPC/Pipe)

  • OpenCodeProcess：定义 start/stop/verifyCommand 方法抽象
  • PosixProcess：POSIX 系统 detached 方式启动，SIGTERM/SIGKILL 优雅终止
  • WindowsProcess：PowerShell 查找并终止子进程树，确保完整回收
  • 支持自定义命令启动与环境变量注入
Section 08

部署与运维

通过热缓存优先、索引辅助、差异追踪与批处理优化，结合生命周期钩子与自动提交，Wiki-Plus 在生产环境中实现了 低 token 成本、高一致性与强可追溯性 的知识管理闭环。

部署流程

1. 克隆仓库
安装 Claude Code、Obsidian、Git，克隆项目仓库
2. 运行初始化脚本
bin/setup-vault.sh 创建 .obsidian 配置、目录结构、CSS 片段
3. 安装社区插件
Calendar、Thino、Excalidraw、Banners、Dataview、Templater、Obsidian Git
4. 多代理适配
bin/setup-multi-agent.sh 将 skills/ 链接到各平台技能目录
5. 验证与使用
输入 /wiki 初始化知识库，开始摄取与查询

性能优化策略

读取策略

  • 优先读取 hot.md (~500 token)，其次 index.md (~1000 token)
  • 使用域子索引聚焦查找，避免全库扫描
  • 控制页面长度 (建议不超过 300 行)

写入策略

  • 差异追踪避免重复摄取
  • 批处理后统一交叉引用与索引更新
  • PATCH 精确编辑，避免整页重写

备份与灾难恢复

建议策略
  • Git 仓库：定期推送至远程仓库，保留历史提交
  • .obsidian/ 配置：纳入版本控制或独立备份
  • 版本回滚：通过 Git 标签管理发布版本，manifest.json 定位变更范围
Section 09

故障排除与常见问题

安装问题

/wiki 命令提示"未找到"
检查插件是否启用，确认在正确目录中打开

Graph 视图颜色重置
打开 Graph 视图 → Color groups → 重新添加，之后永久生效

Excalidraw 加载失败
运行 setup-vault.sh 下载 main.js (约 8MB)

功能异常

热缓存未加载
检查 hooks 列表，确认 SessionStart 钩子存在且 hot.md 格式正确

MCP 无法连接
确认 Local REST API 插件已安装，验证端点返回 JSON

自动备份未触发
检查 Obsidian Git 插件设置与 PostToolUse 钩子

紧急情况应急处理

紧急情况处理方案
热缓存损坏删除 wiki/hot.md，重新执行"更新热缓存"命令
MCP 连接失败降级使用 REST API，检查 Authorization 头与端点
自动备份中断手动执行 git add/commit，检查插件设置
图形视图异常重置 Graph 视图设置的颜色组并重新添加
