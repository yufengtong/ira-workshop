# 10 — 数据模型与存储规格

| 属性 | 内容 |
|------|------|
| **模块编号** | M4 |
| **模块名称** | 团队协作 · 舆情/趋势与多渠道推送 |
| **文档包版本** | v1.2 |
| **技术栈** | 前端 React · 后端 Flask · 代码 GitHub |
| **开发方式** | CoPaw + 本模块 Spec |
| **模块侧重** | 消息获取 + 舆情/趋势规则 + 钉钉/飞书/邮件；Git 分支分工与合并规范。 |

---

## 1. 概念模型

```
notify_channel_configs 1 ── * notify_rules（多对多可经 notify_rule_channels）
notify_message_templates 1 ── * notify_rules（可选 FK template_id）
notify_rules 1 ── * notify_deliveries
```

---

## 2. 表定义（PostgreSQL）

### 2.1 `notify_channel_configs`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `UUID` | PK，`channelId` |
| `type` | `VARCHAR(16)` | dingtalk/feishu/email |
| `label` | `VARCHAR(128)` | |
| `enabled` | `BOOLEAN` | |
| `secret_ref` | `VARCHAR(256)` | Webhook URL / App Secret **引用**，禁止明文 |
| `config` | `JSONB` | 非敏感：机器人 id、邮箱 from 等 |
| `last_test_at` | `TIMESTAMPTZ` | |
| `last_test_status` | `VARCHAR(16)` | |
| `created_at` | `TIMESTAMPTZ` | |
| `updated_at` | `TIMESTAMPTZ` | |

### 2.2 `notify_message_templates`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `UUID` | PK |
| `name` | `VARCHAR(128)` | |
| `channel_type` | `VARCHAR(16)` | |
| `subject` | `VARCHAR(256)` | 可空 |
| `body_markdown` | `TEXT` | |
| `version` | `INT` | NOT NULL，默认 1 |
| `created_at` | `TIMESTAMPTZ` | |

### 2.3 `notify_rules`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `UUID` | PK |
| `name` | `VARCHAR(128)` | |
| `enabled` | `BOOLEAN` | |
| `trigger_type` | `VARCHAR(24)` | |
| `schedule_cron` | `VARCHAR(64)` | NULL |
| `condition` | `JSONB` | |
| `template_id` | `UUID` | NULL，FK |
| `created_at` | `TIMESTAMPTZ` | |
| `updated_at` | `TIMESTAMPTZ` | |

### 2.4 `notify_rule_channels`（多对多）

| 字段 | 类型 | 说明 |
|------|------|------|
| `rule_id` | `UUID` | PK 复合 |
| `channel_id` | `UUID` | PK 复合 |

### 2.5 `notify_deliveries`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `UUID` | PK，`deliveryId` |
| `rule_id` | `UUID` | NULL |
| `channel_id` | `UUID` | NOT NULL |
| `status` | `VARCHAR(24)` | |
| `dry_run` | `BOOLEAN` | NOT NULL |
| `payload_preview` | `TEXT` | 截断 |
| `payload_hash` | `VARCHAR(64)` | 去重可选 |
| `error_code` | `VARCHAR(64)` | NULL |
| `trace_id` | `VARCHAR(64)` | |
| `source_ref` | `VARCHAR(128)` | 预留 M1 messageId |
| `compliance_scan_id` | `VARCHAR(64)` | 预留 |
| `created_at` | `TIMESTAMPTZ` | |

**索引**：`(channel_id, created_at DESC)`；`(rule_id, created_at DESC)`。

## 3. 频控（实现建议）

- 表 `notify_rate_buckets`（P1）或 Redis sliding window；**上限** 与 `07` 一致。  
- P0 可在应用内存 + 简单按用户/渠道计数。
