| 属性 | 内容 |
|------|------|
| **模块编号** | M4 |

---

## 1. 逻辑架构

- React SPA ↔ Flask API（`BASE=/api/v1`）↔ 本模块领域服务 ↔ 外部依赖（数据源/向量/IM Webhook 等）。  
- 标出与 **兄弟模块** 的边界：订阅 M1/M2 事件或轮询 API；模板与频控与全局用户权限对齐。

## 2. 技术选型

| 层 | 选型 | 说明 |
|----|------|------|
| 前端 | React | 与全项目一致 |
| 后端 | Flask | Blueprint 按域拆分 |
| 持久化 | PostgreSQL（建议） | RDS |
| 对象 | OSS（若本模块需要） | 存文件/快照 |

## 3. 部署与环境

- `dev` / `staging` / `prod`；配置来自环境变量与 GitHub Secrets。

## 4. CoPaw 边界

见全项目 [`00` §7](../../00-全项目-企业研发规范-Mock.md)。本模块：**Channel** 出站与 **Skill** 调 BFF 为推荐组合；**合规扫描、模板版本、发送审计** 必须在 BFF/DB 落地，不得仅存在于 CoPaw 内存。
