## 1. 背景与问题

投研信号需 **及时触达** IM/邮箱；须 **合规扫描、频控、审计** 与多渠道适配。Git 协作规范由团队另行遵守（见 `00`）。

## 2. 提案概要

交付 **notify** 域 API + 运维 UI；**默认 dry-run** 安全策略；CoPaw **Channel** 可选双写（`README`）。

## 3. 范围边界

**范围内**：`09`/`10` P0（规则、dispatch、deliveries、channel test）。  
**范围外**：未对接真实合规中台前可用 stub；M1 事件订阅可后置。

## 4. 成功标准

- `05` P0 可测；`13`/`14` 映射。  
- 至少 **1** 条沙箱投递记录。

## 5. 风险

| 风险 | 应对 |
|------|------|
| 渠道封禁 | 多通道备份、降频（`07`） |
| 合规漏扫 | 强制 dispatch 前扫描钩子 |
