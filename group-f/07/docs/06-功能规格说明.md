# 06 — 功能规格说明

| 属性 | 内容 |
|------|------|
| **模块编号** | M4 |
| **模块名称** | 团队协作 · 舆情/趋势与多渠道推送 |
| **文档包版本** | v1.2 |
| **技术栈** | 前端 React · 后端 Flask · 代码 GitHub |
| **开发方式** | CoPaw + 本模块 Spec |
| **模块侧重** | 消息获取 + 舆情/趋势规则 + 钉钉/飞书/邮件；Git 分支分工与合并规范。 |

---

## 1. 边界

- **P0**：规则 CRUD、`dispatch`（默认 **dryRun=true**）、投递历史、渠道连通性测试；合规扫描可 **stub** 返回通过，但字段 `compliance_scan_id` 预留。  
- **M1/M2 事件触发**：预留 `triggerType=event` 与 `sourceRef`，接线在集成期完成。

## 2. 前端路由

| 路由 | 职责 |
|------|------|
| `/notify/rules` | 规则列表与新建/编辑 |
| `/notify/dispatch` | 试发表单：`channelIds`、`payload`、`dryRun` 显式开关 |
| `/notify/history` | `deliveries` 表 |
| `/notify/channels` | 渠道配置与 **test** |

## 3. 状态与交互

- 真发前 **二次确认**；拦截展示 `M4_COMPLIANCE_BLOCK` 详情（可简化）。  
- 与 `09` DTO 一致。

## 4. 后端

- Blueprint：`notify_bp`。  
- 模板变量渲染失败 → **400**，不发出半渲染内容。

## 5. Git 协作

分支策略见模块内专项说明或 `00`；**不**写入本 `06` 细节以免与代码仓库漂移。
